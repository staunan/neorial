Table Borders

To specify table borders in CSS, use the border property.

.table1, .table1 th, .table1 td{
	border: 1px solid black;
}

Notice that the table in the example above has double borders. This is because we have applied the border style on the table element, the <th> element and <td> element. This is why each element draw its own border. To overcome this issue, we can use property border-collapse.

Example: Horizontal Dividers

Instead of using border like before, add the border-bottom property to <th> and <td> for horizontal dividers:

.table{
	border-collapse: collapse;
	width: 100%;
}
.table th{
	height: 50px;
	padding: 10px;
	border-bottom: 1px solid #eee;
}
.table td{
	padding: 10px;
	border-bottom: 1px solid #eee;
}

border-collapse

The border-collapse property sets whether the table borders should be collapsed into a single border. It has two values:

  1. collapse: The adjacent borders will collapse into single border.
  2. separate: This is default value, it means the border won't collapse. It will let the element have their own border.

This property should only be applied on table element, not td or th or tr elements.

.table2{
	border-collapse: collapse;
}
.table2, .table2 th, .table2 td{
	border: 1px solid black;
}

Width and Height

Width and height of a table elements are defined by the width and height properties. These properties can be applied on all table elements like table, th, td, tr, tbody, tfoot, caption etc.

The example below sets the width of the table to 100%, and the height of the <th> elements to 50px:

.table3{
	border-collapse: collapse;
	width: 100%;
}
.table3, .table3 th, .table3 td{
	border: 1px solid black;
}
.table3 th{
	height: 50px;
}

Note that in the above example, we have applied height 50px to table header only. But inner table cell also has the same height even though we haven't applied any height to td element. This is because we have th element in front of every row. So if we set any height to th, it increases the row height as well, and making all td element of that row size.

Horizontal Alignment

The text-align property sets the horizontal alignment (like left, right, or center) of the content in <th> or <td>.

By default, the content of <th> elements are center-aligned and the content of <td> elements are left-aligned. In the following example, we are aligning content to center of all td cell:

.table4{
	border-collapse: collapse;
	width: 100%;
}
.table4, .table4 th, .table4 td{
	border: 1px solid black;
}
.table4 th{
	height: 50px;
}
.table4 td{
	text-align: center;
}

By default, the vertical alignment of the content in a table is top (for both <th> and <td> elements). In the following example, we are setting vertical alignment of all td cell to bottom:

Responsive Table

.table th{

border-spacing

The border-spacing property sets the distance between the borders of adjacent cells. Here are some rules to keep in mind while using border-spacing:

  1. This property only works on table element.
  2. The value specifies the distance between the borders of adjacent cells in px, cm, etc.
  3. Negative values are not allowed.
  4. You can provide two value instead of one value: If one value is specified, it defines both the horizontal and vertical spacing between cells. If two values are specified, the first sets the horizontal spacing and the second sets the vertical spacing.
  5. This property works only when border-collapse property is seperate.
.table, .table th, .table td{
	border: 1px solid black;
}
.table{
  border-collapse: separate;
	border-spacing: 20px;
}

caption-side

The caption-side property specifies the placement of a table caption. It can have one of the following two values:

  1. top: Puts the caption above the table. This is default.
  2. bottom: Puts the caption below the table

empty-cells

The empty-cells property sets whether or not to display borders on empty cells in a table.

The empty-cells property in CSS selects empty table cells for the purpose of specifying whether or not to display borders and backgrounds on them. In other words, it tells the browser whether or not to draw borders around a table cell or fill in the background when that cell contains no content. It's kind of like applying a visibility property on empty table cells. It can take one of the following two values:

  1. hide: Display a border and background on an empty cell.
  2. show: Does not display a border or background on an empty cell.

This property has no effect if border-collapse is "collapse".

table {
  empty-cells: show;
}

A good use case for empty-cells could be a situation where you may not know whether a table will or will not contain empty data points and you decide to hide them. Tables used to be the de-facto way webpage layouts were constructed, so it could be a useful tool for showing and hiding elements when tables are used for presentation or where elements contain the display: table; property.

Here is an example:

.table14{
  caption-side: top;
  empty-cells: hide;
  border-collapse: separate;
  border-spacing: 4px;
}
.table14, .table14 td, .table14 th{
  border: 1px solid #ddd;
}
.table14 td, .table14 th{
  background: #eee;
}

table-layout

The table-layout property defines what algorithm the browser should use to lay out table rows, cells, and columns.

It can accept either of two values:

  1. auto: The default. The browser's automatic algorithm is used to define how a table's rows, cells, and columns are laid out. The resulting table layout is generally dependent on the content of the table.
  2. fixed: With this value, the table's layout ignores the content and instead uses the table's width, any specified width of columns, and border and cell spacing values. The column values used are based on widths defined on columns or cells for the first row of the table. If you don't specify width for each column in the table, then each column will have equal width to fill the width of the table.

Example: auto

.table15{
  caption-side: top;
  table-layout: auto;
  width: 100%;
}
.table15, .table15 td, .table15 th{
  border: 1px solid #ddd;
}
