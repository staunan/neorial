Accounts

Accounts within Solana are used to store state. They are an essential building block for developing on Solana. Here is few characterstics:

  • Accounts are used to store data
  • Each account has a unique address
  • Accounts have a max size of 10MB (10 Mega Bytes)
  • PDA accounts have a max size of 10KB (10 Kilo Bytes)
  • PDA accounts can be used to sign on behalf of a program
  • Account size is static
  • Account data storage is paid with rent
  • Default account owner is the System Program

Set Configuration

RPC URL : Update Cluster URL to Devnet

To make sure you are on the devnet, type the following command:

solana config set --url https://metaplex.devnet.rpcpool.com/

This will show output like the following:

Config File: C:\Users\NeoRe\.config\solana\cli\config.yml
RPC URL: https://metaplex.devnet.rpcpool.com/
WebSocket URL: wss://metaplex.devnet.rpcpool.com/ (computed)
Keypair Path: C:\Users\NeoRe\.config\solana\id.json
Commitment: confirmed

To know the cluster version, use:

solana cluster-version

Solana-Keygen

To work with wallet, we need to use solana-keygen command. This tool is installed by default with solana-cli installation.

To know the solana-keygen version, use:

solana-keygen --version

New Wallet : File System Wallet

The solana's config file is located in the directory C:\Users\<username>\.config\solana. Lets create a new folder inside this directory called wallets under which we will keep all our wallets here. Now run the following command to create a new wallet in that directory:

solana-keygen new -o C:\Users\NeoRe\.config\solana\wallets\account_1.json

This will prompt you to enter BIP39 Passphrase. You can skip this. The above command will generate a new file inside the wallets directory. This file contains unencrypted keypair of your wallet. The wallet has been created successfully, but it is not assigned with the Solana-CLI yet. The next step is to make this file our default keypair. Enter the following command to make the file as our default keypair/wallet:

solana config set --keypair C:\Users\NeoRe\.config\solana\wallets\account_1.json

Now you can run following command to know the address or public key associated with the default account, which is in our case account_1

solana address

solana pubkey

The above command solana pubkey display the public key of default account, to know the public key of any account use the file path:

solana pubkey <C:\Users\NeoRe\.config\solana\wallets\account_2.json>

To know the balance, use the following command:

solana balance

This way you can create wallet as many as you want and make any wallet as your default wallet.

Fund Wallet | Solana Faucet

In order to add SOL to your devnet wallet, you can request funds from a faucet:

solana airdrop 2

If the command is successful, you will see the updated balance at the end. Make sure you are entering the airdrop amount that is within the airdrop limit. Currently the maximum airdrop request limit is 2 SOL and there is a daily total limit of 24 SOL.

Run solana balance to check your wallet is funded with 2 SOL.

Lamports

A SOL is the name of Solana's native token, which can be passed to nodes in a Solana cluster in exchange for running an on-chain program or validating its output. The system may perform micropayments of fractional SOLs, which are called lamports. They are named in honor of Solana's biggest technical influence, Leslie Lamport. A lamport has a value of 0.000000001 SOL
