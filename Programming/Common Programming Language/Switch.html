Switch

A switch statement can replace multiple if checks. It gives a more descriptive way to compare a value with multiple variants. The syntax of the switch statement looks like this -

switch(value){
	case value1 : 
		statements...
	case value2 :
		statements...
	case value3 :
		statements...
	...
	default:
		statements...
}

In the switch statement we have value. That value can be an expression, or a function call that returns value, or it can be a variable or constant. However, the final result of the value is considered. When the control is inside the switch statement, the value is checked against each value that is given to the case statements. If any case statement matches with the switch value, then that case statement gets executed even after that all the following case statements also get executed including the default statement. When no case value matches with the switch value, then the default statements gets executed.

Here is an example -

let x = 30;
switch(x){
	case 10:
		console.log("Ten");
	case 20:
		console.log("Twenty");
	case 30:
		console.log("Thirty");
	case 40:
		console.log("Fourty");
	case 50:
		console.log("Fifty");
	default:
		console.log("Unknown");
}

And here is the output -

Thirty
Fourty
Fifty
Unknown

When the control picks up the value from the switch statement, the value is checked with the first case value, as it doesn't match, the control then check the value with the second case statement, in this case also, the value doesn't match. So control compares the value with the third case value. It matches with the switch value. So control executes all the statement that is under that case statement. But not only that, after finishing all the statements of that case, the control executes all the statement of all the following case statement without checking their values. Even the default statements gets executed too. That means once a case value matches with the switch value, all the statement after that gets executed.

Don't enclose multiple statements within brackets

Even if you have multiple statement to execute, don't wrap them within the curly braket. Just put them under the case one after another. For example -

// Wrong
switch (value){
	case value:
		{
			statement1;
			statement2;
			statement3;
		}
		...
}

// Correct
switch (value){
	case value:
		statement1;
		statement2;
		statement3;
		...
}

Switch Statement with Break

The break statement can be used with the switch statement. When the control hits the break statement, the control immediately breaks out of the switch statement. And that logic can be used to prevent execution of subsequent case and default statements. Consider the above example, in the following example, we are preventing the control from getting into other case statement after finishing execution of the matched case statements.

let x = 30;
switch(x){
	case 10:
		console.log("Ten");
		break;
	case 20:
		console.log("Twenty");
		break;
	case 30:
		console.log("Thirty");
		break;
	case 40:
		console.log("Fourty");
		break;
	case 50:
		console.log("Fifty");
		break;
	default:
		console.log("Unknown");
		break;
}

And the output will be -

Thirty

Grouped Case statements

Using the break statement, several Case statement can be grouped together. Here is an example -

let a = 2 + 2;

switch (a) {
  case 4:
    alert('Right!');
    break;

  case 3:  // (*) grouped two cases
  case 5:
    alert('Wrong!');
    alert("Why don't you take a math class?");
    break;

  default:
    alert('The result is strange. Really.');
}

Now both 3 and 5 show the same message.
