xTextBox

let myTextbox = dom.createXElement(xTextBox, {
	type: "xTextBox",
	counter: "45",
	prevent-overflow: true,
	required: true,
	disabled: true,
});

Represents a TextBox. This component is built on top of xInputWrapper.

Attributes

counter

Specifies the maximum number of character that can be allowed in the textbox. It also shows the counter.

prevent-overflow

If this attribute is true, then user cannot type more than maximum character that was specified by counter property.

required

Specifies that the field is required when used inside xForm. If true, whenever the textbox looses focus the component validates the data and shows error message if the value is empty.

disabled

The textbox will be blocked from interacting and visually will be grayed to show it is disabled input.

help-text

It takes a string that will be used to set the help text. Whenever the textbox gains focus, the help text becomes visible. Help text is shown in place of error message. If there is an error, then error message will take the priority.

format

The type of text that can be allowed in the text box. Currently it supports two value:

  • numeric: Allows numeric value.
  • urlencodestring: Text box for entering url.

Without this attribute any text can be entered into the textbox.

autocomplete

By default the browser autofills the textbox. You can override this nature of the browser by passing an autocomplete attribute and giving the value off.

placeholder

Specifies the placeholder of the textbox. The placeholder is shown on over the editable area. If you provide both the label and placeholder, then both will be visible and overlap each other. This bug will be fixed very soon. The placeholder is useful when you don't need a label.

You can provide the following attributes which are simply passed to the component xInputWrapper:

  • expand-collapse-icon
  • icon
  • label
