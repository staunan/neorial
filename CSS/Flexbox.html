What is Flexbox

Flexbox is amazing thing happened to CSS. It's a layout system. Which is flexible and supported in all major browser. It's native to CSS. Using flexbox layout system you can do much more than you can do with position, float stuff. By default a div is a block level element. It means its display is block by defult. If you have multiple div one after another they apeared vertically one after another. This default behaviour can be changed by using flexbox.

Consider the following CSS and boxes which we are going to use through out the chapter:

.Red{
	min-height: 50px;
	min-width: 50px;
	color : #fff;
	background-color: red;
}
.Blue{
	min-height: 50px;
	min-width: 50px;
	color : #fff;
	background-color: blue;
}
.Green{

	background-color: green;

	background-color: yellow;

	color : #fff;
	background-color: grey;
}

	min-width: 50px;
	color : #fff;    		
	background-color: tomato;

flex-basis

It specifies the initial size of the flex item, before any available space is distributed to the childs. If you set the flex-basis, initially an item will take that space in the flexbox container. So, this is kind of specifying the width of the container.

Example: One

Let's consider an example where parent element has a specific width. All child element can perfectly fit inside the parent element's width, even after specifying flex basis. That means the parent has enough space to accomodate all children with their respective flex-basis values. In this case the element will take exactly that much width what is specified on property flex-basis. For example, let's specify the Red block's flex-basis:

<div class="d-flex" style="width: 1000px; background: grey;">
	<div class="Red" style="flex-basis: 100px;">Child-1</div>
	<div class="Blue">Child-2</div>
	<div class="Green">Child-3</div>
	<div class="Tomato">Child-4</div>
</div>

In the example above, The first red box is given 100px width and positioned. And then the next blue element is positioned. And then green and so on. As other color boxes do not have flex-basis specified, they will take whatever their content requires. If the parent element requires more space to accomodate all its children, then it will shrink flex items with to fit all children.

In the above example, the parent element is greyed background which is 1000px wide. Only the red box is having flex-basis of 100px. And other boxes do not have any flex-basis specified, so other colored boxes will take space whatever their content requires. As there is enough space on the parent element, the red box will take 100px width to satisfy the flex-basis.

Note that, it is important to set the width to the parent element in order for flex-basis to take effect.

flex-wrap

The flex-wrap property specifies whether the flexible items should wrap or not. It defines whether the flex items are forced in a single line or can be flowed into multiple lines. The flex-wrap property can have the following three values:

  1. wrap
  2. wrap-reverse
  3. nowrap

stretch

This is the default value if you don't use align-items property. This value stretches the item so that the blocks takes the full height of the Container. If you notice in the justify-content examples, all the blockes are stretched and taking full height. This is because we were not using align-items property. flex-grow

It defines the ability for a flex item to grow if necessary. It accepts a unitless value that serves as a proportion. It dictates what amount of the available space inside the flex container the item should take up.

Example One: Equal width when flex-grow is 1

For example, if all items have flex-grow set to 1, every child will set to an equal size inside the container. If you were to give one of the children a value of 2, that child would take up twice as much space as the others.

<div class="d-flex width-300">
	<div class="Red flex-grow-1"></div>
	<div class="Blue flex-grow-1"></div>
	<div class="Tomato flex-grow-1"></div>
</div>

Example Two: When flex-grow is 2

<div class="d-flex width-300">
	<div class="Red flex-grow-1"></div>
	<div class="Blue flex-grow-2"></div>
	<div class="Tomato flex-grow-1"></div>
</div>

Notice, the blue box is taking twice space as the other as its flex-grow is set to 2. All items have a flex-grow value of 1 except the second one which has a flex-grow value of 2. It means when the available space is distributed, the second flex-item will have twice as much space as others.

Example Three: Using flex-grow with flex-basis

The property flex-grow can be work together with flex-basis to control the width of blocks. For example, your parent container has the width 500px, and you want to have two blocks which has the flex-basis 230px and 200px. These two blocks certainly can fit within the parent container. But it would leave extra 70px of space empty. If you want to distribute the remaining space within them use flex-grow property to these blocks. The width of these two blocks will increase according to the value of flex-grow. For example, if your first block's flex-grow is 3 and the second block's flex-grow is 2, then the remaining space will be distributed with the ratio 3:2.

<div class="d-flex width-500">
	<div class="Red flex-grow-3" style="flex-basis: 230px;"></div>
	<div class="Blue flex-grow-2" style="flex-basis: 200px;"></div>
</div>

flex-shrink

It specifies the "flex shrink factor", which determines how much the flex item will shrink relative to the rest of the flex items in the flex container when there isn't enough space on the row.

<div class="d-flex width-500" style="background-color: green;">
	<div class="Red" style="flex-basis: 430px;"></div>
	<div class="Blue" style="flex-basis: 300px;"></div>
</div>

In the above example, total we have 500px and within it we are trying to accomodate a div which has width of 730px. In the above example flex-shrink is not applied so by default it is set to 1. It means all the blocks within the container will shrink by the same amount of width to fit within its container. In the following example, we set flex-shrink to 0. It means there will no shrinkage. Thus all the block will take the specified width and it will cause overflow.

In the following example, we set flex-shrink with different amount. The second block is set to 1 and first block is set to 0.

As you can see in the above example, the second block will shrink alone and take 70px width as the first block will take 430px because its flex-shrink is 0.

flex-flow

It is a shorthand for flex-direction and flex-wrap. You can specify one or two values, no matter the order.

<div class="d-flex w-500" style="flex-flow: wrap row;">
	<div class="Red width-100 m-10"></div>
	<div class="Blue width-200 m-10"></div>
	<div class="Green width-50 m-10"></div>
	<div class="Tomato width-500 m-10"></div>
	<div class="Blue width-400 m-10"></div>
	<div class="Green width-300 m-10"></div>
	<div class="Red width-50 m-10"></div>
</div>

In the above example, the first value specifies the value for flex-direction and second value is used for flex-wrap property. The values can be appreared in reversed oder. Also you can provide a single value as well. The missing value will have default value.

align-self

It makes possible to override the align-items value for specific flex items.

<div class="d-flex height-500 align-items-start flex-wrap" style="background-color: grey;">
	<div class="Red align-self-center"></div>
	<div class="Blue align-self-end"></div>
	<div class="Green align-self-baseline">Hello World</div>
	<div class="Tomato align-self-stretch"></div>
	<div class="Blue align-self-end"></div>
	<div class="Green align-self-center"></div>
	<div class="Red"></div>
</div>
