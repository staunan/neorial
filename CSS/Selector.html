CSS Selectors

CSS selectors are used to "find" (or select) the HTML elements you want to style. We can divide CSS selectors into five categories:

  1. Simple selectors (select elements based on name, id, class)
  2. Combinator selectors (select elements based on a specific relationship between them)
  3. Pseudo-class selectors (select elements based on a certain state)
  4. Pseudo-elements selectors (select and style a part of an element)
  5. Attribute selectors (select elements based on an attribute or attribute value)

Class selectors (.className)

To select an element with class name use .className. For example, the class boxy gets applied to the element who's one of the class contains boxy.

.boxy{
	width: 100px;
	height : 100px;
	background-color : red;
	margin : 10px;
	display : inline-block;
}

<div class="box">
	<div class="boxy"></div>
	<div class="boxy"></div>
</div>

ID selectors (#idName)

To select an element with the id name, use #idName syntax.

#boxy1{
	width: 100px;
	height : 100px;
	background-color : red;
	margin : 10px;
	display : inline-block;
}
#boxy2{
	width: 100px;
	height : 100px;
	background-color : blue;
	margin : 10px;
	display : inline-block;
}

<div class="box">
	<div id="boxy1"></div>
	<div id="boxy2"></div>
</div>

The universal selector (*)

Selects all element in the document. The symbol * is used to specify all elements. For example, the following css gets applied to all elements in the document.

*{
	font-size : 15px;
	box-sizing : border-box;
}

The following example shows how to select all element inside a class:

.boxy *{
	font-size : 15px;
	box-sizing : border-box;
}

The following example shows how to select all element inside an element with id:

#idName *{
	font-size : 15px;
	box-sizing : border-box;
}

Element Selectors (element)

You can select an element by its element name. For example to color all paragraph, you can use following code:

p{
	font-weight: bold;
	font-size : 19px;
	color : blue;
}

To color heading use the following code:

h1{
	font-weight: bold;
	font-size : 19px;
	color : blue;
}

selector1.selector2

When two selector is combined with a single dot operator, then both selector should be present on the same HTML element to be able to match.

For example, the following example will match those h1 element whose one of the classname is fontStyle.

h1.fontStyle{
	color : red;
}

For example, the following example will match those element which has both class expired and fontStyle

.expired.fontStyle{
	color : red;
}

For example, the following example will match those element which has id expired and class fontStyle

#expired.fontStyle{
	color : red;
}

selector1, selector2

If you want to apply some common css rules to different selectors, you can use multiple selectors seperated by comma.

Example, the following css rules will be applied to both class red_text and danger_text:

.red_text, .danger_text{
	color : red;
}

In the example below, all paragraph and heading will have margin bottom of 10px:

	
p, h1{
	margin-bottom : 10px;
}

selector1 selector2

When two selectors are seperated by space, then the second selector should be the child of first selector, in order to be able to match. This type of selector is called, Descendant Selector. The word Descendant indicates nested anywhere in the DOM tree. It can be a direct child or deeper than five levels, but it will still be referred to as a descendant.

For example, the following example will select element with class element2 and it must be child of class element1

.element1 .element2{
	line-height : 18px;
}

In the following example, all the p element will be selected whose ancestor div element has class hello.

div.hello p{
	line-height : 18px;
}

selector1 > selector2

This is also called Child Selector. Elements matched by the second selector must be the immediate children of the elements matched by the first selector.

In the following example, it selects all element with class element2 and its immediate parent class is element1.

.element1 > .element2 { 
	background-color: yellow;
}

The following example will match those paragraph p element, which are direct children of div element.

div > p { 
	background-color: yellow;
}

selector1 + selector2

This is also called Adjacent Sibling Selector. Elements with selector2 must be immediate sibling of selector1. That means selector1 element should come first and then selector2 element.

The following example will match those element which has class .element2 and its previous element has class element1.

.element1 + .element2{ 
	// CSS rules
}

The following example will match those p element whose previous sibling element is h1. That means, it will select all the first paragraph after the h1 element.

h1 + p { 
	background-color: yellow;
}

selector1 ~ selector2

This is also called General Sibling Selector. The selector1~selector2 selector matches occurrences of selector2 that are preceded by selector1. Selects all selector2 that are preceed by the selector1. Both elements must have the same parent, but selector2 does not have to be immediately preceded by selector1.

element1 ~ element2{
	background : red;
}

The following example will make the background color red to both ul element.

p~ul{
	background-color : red;
}

<p>The first paragraph.</p>
<ul>
	<li>Coffee</li>
	<li>Tea</li>
	<li>Milk</li>
</ul>

<h2>Another list</h2>
<ul>
	<li>Coffee</li>
	<li>Tea</li>
	<li>Milk</li>
</ul>

[attribute]

The [attribute] selector is used to select elements with the specified attribute.

The following example selects all the element whose one of the attribute is href.

[href]{
	color : yellow;
}

The following example selects all the element whose one of the attribute is target.

[target]{
	color : red;
}

[attribute=value]

The [attribute=value] selector is used to select elements with the specified attribute and value.

[attribute = value] {
	css declarations;
}

The following example matches element with target='_blank' attribute.

[target=_blank] {
	color : green;
}

The following example selects all the input element whose one of the attribute is type and it has the value text.

input[type=text] {
	width: 100px;
	transition: ease-in-out, width .35s ease-in-out;
}
input[type=text]:focus {
	width: 250px;
}

Combinators

A CSS combinator is a type of selector in CSS which is used to specify the relationship between exactly two CSS selectors. It essentially combines two CSS selectors to uniquely identify an element in a document.

There are four different combinators in CSS:

  1. descendant selector (space)
  2. child selector (>)
  3. adjacent sibling selector (+)
  4. general sibling selector (~)

Specificity

If there are two or more conflicting CSS rules that point to the same element, the browser follows some rules to determine which one is most specific and therefore wins out. Think of specificity as a score/rank that determines which style declarations are ultimately applied to an element. The universal selector (*) has low specificity, while ID selectors are highly specific!

Specificity is a common reason why your CSS-rules don't apply to some elements, although you think they should.

Every selector has its place in the specificity hierarchy. There are four categories which define the specificity level of a selector:

  1. Inline styles An inline style is attached directly to the element to be styled. Example: <h1 style="color: #ffffff;">.
  2. IDs An ID is a unique identifier for the page elements, such as #navbar.
  3. Classes, attributes and pseudo-classes This category includes .classes, [attributes] and pseudo-classes such as :hover, :focus etc.
  4. Elements and pseudo-elements This category includes element names and pseudo-elements, such as h1, div, :before and :after.

In the following example, the background color pink will take the priority.

#one{
	background-color : pink;
}
.one{
	width: 300px;
	background-color : red;
	float  : left;
	height : 300px;
}

<div id="one" class="one"></div>

Equal Specificity

If the same rule is written twice into the external style sheet, then the lower rule in the style sheet is closer to the element to be styled, and therefore will be applied. In the following example, background color red gets applied. Which means, the last one gets applied.

h1 {background-color: yellow;}
h1 {background-color: red;}

In the following example, the first rule is more specific than the other two, and will be applied.

div#a {background-color: green;}
#a {background-color: yellow;}
div[id=a] {background-color: blue;}

Different File

If your external style sheet contains the same selector as in your main file where you are importing the external file, then the one which is defined in the main file will be used, and the same style from the external file will be ignored. In the example below, background color yellow will take the priority and will be applied.

// From external CSS file:
#content h1 {background-color: red;}

// In HTML file:
<style>
#content h1 {
	background-color: yellow;
}
</style>
