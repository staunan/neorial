Box Model

Every element in web design is a rectangular box. Box Model specifies how CSS calculates a box's width and how you can change the default behaviour. The following picture demonstrate how border, padding and content are positioned within a box.

Following is the common formula for calculating total with of a box:

  1. Total Width : content width + padding-left + padding-right + border-left + border-right
  2. Total Height: content height + padding-top + padding-bottom + border-top + border-bottom

The property box-sizing allows us to specify the behaviour of how our code calculates the with of a box. It has two values:

  1. border-box
  2. content-box

If you apply box-sizing property, the behaviour applies to all child element inside it, unless any child element has it's own box-sizing value specified.

border-box

In border-box, if you don't declare a width of a box element, and the box has static or relative positioning, the width will remain 100% in width and the padding and border will push inwards instead of outward. To demonstrate how border-box behaves, consider the following example:

  1. The first div do not have width specified. So the default width is 100%. The padding and border width pushes inwards, so the actual with of the content area is less than 100%.
  2. The second div is set with width 100%. And it is same as previous div.
  3. The third div is set with width 800px, but actual content area for paragraph is less than 800px. Because within 800px, there's left and right border width, and left and right padding is calculated. As padding is 50px and border width is 10px, the total width for content area would be 800px - (50px + 50px + 10px + 10px).
  4. The fourth div is a 800px line to demonstrate the masurment of the previous divs.

As you can see in the above examples, in border-box, the specified width also includes padding width and border width.

content-box

This value is the default value. In this behaviour, the padding and border width pushes outwards of the specified width. For example, if you have a div of 100px and there's a padding of 10px and border of 10px on both left and right side, then total width for that box would be (100px + 10px + 10px + 10px + 10px) = 140px. To demonstrate the behaviour let's consider the same example, but this time the parent div contains box-sizing: content-box;.

  1. The first div has no width specified. Without width specified, the div behaves like border-box. That means the padding and border width pushes inwards instead of outwards.
  2. The second div has width specified of 100%. But the padding and border makes the box bigger than 100% of its size. Because if you specify width in content-box behaviour, the padding and border width will always increase the box width.
  3. Similarly, for third box, the width 800px is specified. But total box width is bigger than 800px as there's padding and border width is added.
  4. Third div is exactly a 800px width, just to compare the previous div to understand the effect.
