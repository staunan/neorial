Setup Go on Windows

Go to the page https://go.dev/doc/install and follow the instruction to install the Golang toolchain. It is a one click installation file.

Setup Go on Linux

Download the setup file from the official page. Then run the following command:

sudo tar -C /usr/local -xzf go$VERSION.$OS-$ARCH.tar.gz

It will unpack and extract the file and create a new folder go under /usr/local directory.

Add Go binary to your path

To work with Go, you need to add path to the environment variable. Run the following command:

export PATH=$PATH:/usr/local/go/bin

Check Version

Run the following command to get the golang version:

C:\Users\NeoRe>go version
go version go1.23.2 windows/amd64

Run a File: Hello World

Create a directory called GoPlayground and change your directory to the newly created directory

mkdir GoPlayground
cd GoPlayground

Under the directory create a file demo.go and paste the following code:

package main

import "fmt"

func main() {
	fmt.Printf("hello, world\n");
}

Then build it with the go tool:

go build demo.go

The command above will build an executable named demo in the current directory alongside your source code. Execute it to see the greeting:

// For Linux
./demo

// For Windows
.\demo

It will produce the following output:

hello, world

Go Syntax: Structure of a GO file

A Go file consists of the following parts:

  1. Package declaration
  2. Import packages
  3. Functions
  4. Statements and expressions

In Go, every program is part of a package. We define this using the package keyword. In the example above, the program belongs to the main package. The package main is entry package when we execute Go code. Execution starts from package main.

import ("fmt") lets us import files included in the fmt package.

func main() {} is a function. Any code inside its curly brackets {} will be executed. The function main is entry function when execution starts.

fmt.Println() is a function made available from the fmt package. It is used to output/print text. In our example it will output "hello, world".

Go Statements

fmt.Printf("hello, world\n"); is a statement.

In Go, statements are separated by ending a line (hitting the Enter key) or by a semicolon ";".

Hitting the Enter key adds ";" to the end of the line implicitly (does not show up in the source code).

The left curly bracket { cannot come at the start of a line. Run the following code and see what happens:

package main
import ("fmt")

func main()
{
	fmt.Printf("hello, world\n")
}

The above code produces the following error:

./prog.go:4:6: missing function body
./prog.go:5:1: syntax error: unexpected semicolon or newline before {

So the correct form of a function body looks like the following where first curly bracket starts after the parenthesis. Its like Go enforcing us to follow a standard way of writing code:

package main
import ("fmt")

func main(){
	fmt.Printf("hello, world\n")
}

Go Comments

A comment is a text that is ignored upon execution. Comments can be used to explain the code, and to make it more readable. Comments can also be used to prevent code execution when testing an alternative code. Go supports single-line or multi-line comments.

Single-line Comments

Single-line comments start with two forward slashes (//).

// This is a comment
package main
import ("fmt")

func main() {
	// This is a comment
	fmt.Println("Hello World!")
}

The following example uses a single-line comment at the end of a code line:

package main
import ("fmt")

func main() {
	fmt.Println("Hello World!") // This is a comment
}

Multi-line Comments

Multi-line comments start with /* and ends with */. Any text between /* and */ will be ignored by the compiler:

package main
import ("fmt")

func main() {
	/* The code below will print Hello World
	to the screen, and it is amazing */
	fmt.Println("Hello World!")
}
